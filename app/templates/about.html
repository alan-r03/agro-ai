About Agro-AI

Background & Motivation

The future of food production is confronted with major challenges, including a declining farming workforce, an aging population of farmers, and increasing global food demand. At present, identifying blighted plants is a manual and time-intensive task, which hampers farming efficiency. ​

The Agro-AI project addresses this issue by leveraging machine learning to automate the detection of plant blight. Its goal is to enhance crop yields while minimizing the need for manual labor. Additionally, the project showcases how artificial intelligence and machine learning can revolutionize agriculture, offering meaningful insights to support farmers and industry professionals.

Main Features

  • Label images as “Healthy” or “Unhealthy” through a simple UI
  • Train/retrain a binary classifier right in your browser session
  • Generate Grad-CAM heatmaps to interpret model decisions
  • Review model confidence and feedback

Architecture & Tech Stack

  • Frontend: Flask + Bootstrap templates
  • Backend: Flask (Python), TensorFlow/Keras
  • Storage: Redis for session & cache
  • Deployment: Unix/Linux host

Development Process

We collaborated on GitHub, used VS Code for daily work, and followed an MVC pattern in Flask. Redis stores intermediate labels, while Keras handles training, and we expose everything via simple Flask routes and Jinja2 views.

Technical Challenges

  • Integrating on-the-fly ML training into a web session
  • Building an intuitive labeling interface
  • Session management & caching with Redis
  • Rendering Grad-CAM overlays cleanly in HTML
